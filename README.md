# 학교 연수 전자서명 배포 원본

다른 학교 선생님이 자기 Google 계정으로 사본을 만들어 독립 운영할 수 있는 빈 배포본입니다. 화면과 백엔드는 학교용 Google 시트에 연결된 Apps Script 웹앱이 함께 제공하며, 명단·연수·서명 기록·서명 이미지는 각 학교 관리 계정의 시트와 비공개 드라이브에 저장합니다.

- [학교용 사본 만들기](https://docs.google.com/spreadsheets/d/1vhVkVvHaAnokOxUd1IxioPtkWg3i1irgeVpuTJtwDjg/copy)
- [배포용 원본과 사용설명서 보기](https://docs.google.com/spreadsheets/d/1vhVkVvHaAnokOxUd1IxioPtkWg3i1irgeVpuTJtwDjg/edit?usp=sharing)

위 첫 번째 링크에서 사본을 만들면 별도 홈페이지를 준비할 필요가 없습니다.

이 프로젝트는 다른 사이트의 코드를 복제하지 않고 새로 구현했습니다. 본인 인증이 필요한 법적 전자서명이나 공식 동의·결재에는 사용하지 마세요.

## 주요 기능

- 연수 선택 → 부서·성명 선택 → 터치·마우스 서명의 3단계 화면
- 날짜·시각·활성 상태와 전체 구성원·선택 부서 대상을 지정하는 여러 연수 관리
- 연수별 선택형 현장 접수 코드: 관리자가 5·10·15분 접수를 열고 현장에서 안내한 6자리 코드로 참석 확인
- 구성원 개별·일괄 등록, 엑셀/CSV 가져오기, 부서명 변경
- 서버와 기기의 중복 제출 방지
- 공유 링크와 QR, 공유 키 즉시 교체
- 관리자 공용 비밀번호, 30분 세션, 로그인 실패 제한
- 연수별 미서명 기본 현황, 전체 서명 상태·완료 시각과 서명률 확인
- 비공개 드라이브 서명 원본, 관리자별 기록 조회·삭제
- 실제 서명 이미지가 포함된 A4 미리보기와 PDF·XLSX·인쇄 선택 출력
- PDF 또는 XLSX 보관 완료 후 원본 일괄 삭제
- `?demo=1` 읽기 전용 미리보기

## 저장소 구조

- `index.html`, `assets/`: 개발·테스트용 화면 소스
- `apps-script/`: Google 시트에 연결해 사용하는 Apps Script 웹앱과 백엔드
- `vendor/`: 고정 버전 QR·엑셀 라이브러리
- `tests/`: 데이터 정규화와 보안 설정 검사
- `OPERATIONS.md`: 설치·운영·복구 설명서

## 로컬 확인

Node.js가 설치된 컴퓨터에서 아래 명령을 실행합니다.

```powershell
npm.cmd test
npm.cmd run check
npx.cmd --yes serve .
```

브라우저에서 표시된 주소 뒤에 `?demo=1`을 붙여 엽니다. 데모 관리자 비밀번호는 `demo-admin`이며 변경 내용은 저장되지 않습니다.

## 배포

일반 사용자는 별도 소스 저장소 계정이나 웹사이트가 필요하지 않습니다. 이 저장소는 개발자가 소스와 테스트를 관리하기 위한 공간입니다.

자세한 순서는 [OPERATIONS.md](OPERATIONS.md)를 따르세요. 학교에서 사용하는 순서는 다음과 같습니다.

1. 전달받은 배포용 Google 시트에서 `파일 → 사본 만들기`로 학교용 사본을 만듭니다.
2. 사본에서 `🖊️ 전자서명 관리 → 초기 설정 실행`을 누르고 본인 Google 계정으로 권한을 승인합니다.
3. `확장 프로그램 → Apps Script → 배포 → 새 배포 → 웹 앱`에서 실행 계정은 `나`, 액세스 권한은 로그인하지 않은 사람도 포함하는 `모든 사용자`로 설정합니다.
4. `/exec` 주소에서 관리자 첫 설정을 마친 뒤 관리자 화면의 공유 링크와 QR을 교직원에게 안내합니다.

## 개인정보 운영 원칙

- 외부 공개 공간과 공유 링크에 명단·서명·비밀번호를 넣지 않습니다.
- 서명 이미지는 공개 공유하지 않습니다.
- 학교가 승인한 관리용 Google 계정과 보관기간을 사용합니다.
- PDF 또는 엑셀을 보관한 뒤 시스템 원본을 삭제합니다. 인쇄창을 연 것만으로는 원본을 삭제할 수 없습니다.
- 공유 링크가 외부로 전달되면 관리자 화면에서 공유 키를 즉시 교체합니다.
- 현장 접수 코드는 참석 여부를 보조 확인하는 수단입니다. 코드를 다른 사람에게 전달하는 행위까지 막는 본인 인증 수단은 아니므로 공식 출석부가 필요한 연수는 함께 대조합니다.

별도 오픈소스 라이선스는 부여하지 않았습니다. 포함된 제3자 라이브러리에는 각 라이선스가 적용됩니다.
